# Dixie Flatline brand positioning

## Summary
Dixie Flatline uses a retro terminal identity for persistent project memory without leaning too hard on source-reference novelty.

## Context
The public launch needs a clear identity that feels developer-first, slightly weird, useful, and dry rather than gimmicky.

## Decision
Position Dixie Flatline as repo-local memory for AI coding assistants with the tagline "Persistent project memory for AI coding agents." Use a dark terminal palette with phosphor green, electric cyan, muted magenta, and amber. Use a flatline waveform turning into code brackets as the primary mark.

## Consequences
The README, package metadata, GitHub description, and generated assets should stay consistent with this positioning.

## Constraints
- Keep copy practical and developer-first.
- Avoid overusing exact Neuromancer references.
- Keep icons readable at small sizes with no tiny text.
- Use the mark by itself for VS Code and GitHub icon contexts.

## Anti-Patterns
- Do not use generic AI purple gradients.
- Do not make the identity horror-themed.
- Do not rely on long in-joke copy to explain the product.
